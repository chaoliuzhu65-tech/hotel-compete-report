# 酒店竞品价格分析 - 技术说明报告

## 项目概述

**hotel-compete-report** 是德胧集团AI-native项目，基于高德地图POI + 飞猪Fliggy双引擎，提供：

1. **多维竞品智能筛选** - v3.1新增动态权重、异常值过滤、TopN精选
2. **AI调价建议引擎** - 价格弹性分析 + 竞争定位指数 + 需求强度三档策略
3. **行业扩展检索** - 酒店周边餐饮/零售/银行/出行全链路配套分析
4. **每日实时监测** - 定时任务自动生成日报推送到飞书

## 核心算法：调价策略详解

### 算法流程

```
Step 1: 价格弹性分析
  → 计算竞品涨幅变异系数（CV）
  → CV > 0.5  = 高弹性（竞争激烈，提价谨慎）
  → CV < 0.2  = 低弹性（价格走势一致，可以激进）

Step 2: 竞争定位指数（CPI）
  → CPI = (目标酒店平日价 / 竞品平均假日价) × 100
  → CPI > 120 = 高端定位 → 可以溢价
  → 90-120   = 中高端定位 → 市场均衡
  → 70-90    = 中端性价比 → 走量为主
  → < 70     = 经济型定位 → 性价比优势

Step 3: 需求强度判断
  → HIGH（节假日/大型展会）→ 选激进策略
  → MEDIUM（周末）→ 选标准策略
  → LOW（工作日/淡季）→ 选保守策略

Step 4: 生成三档定价
  → 基于竞品涨幅中位数
  → 保守：中位数 × 0.85
  → 标准：中位数 × 弹性调整系数
  → 激进：中位数 × 1.20
```

### 计算公式

**价格弹性（变异系数CV）**
```
CV = 标准差(竞品涨幅) / 平均值(竞品涨幅)
```
CV越大，价格分散程度越高，说明市场竞争越激烈。

**竞争定位指数CPI**
```
CPI = (目标酒店基准价 / 竞品平均假日价) × 100
```
CPI反映了你的价格在市场中的相对位置。

**最终推荐价格**
```
推荐价格 = 基准价 × (1 + 调整后中位数涨幅 / 100)
```
调整系数 = 1 + (1 - 弹性得分) × 0.15 × 需求得分
- 低弹性 + 高需求 → 系数更大 → 更大涨幅

## v3.1 竞品选择逻辑改进

### 原问题

v3.0及之前：
- 固定权重 30%/30%/20%/20% 不能适应不同场景
- 没有异常值过滤，满房天价/促销超低价干扰结果
- 不考虑occupancy信号（竞品满房是重要提价机会）
- 所有高分竞品都保留，太多竞品稀释信号

### v3.1 改进方案：分层筛选

```
1️⃣ 第一层：距离硬过滤
   → 超过max_distance_km直接排除

2️⃣ 第二层：星级匹配过滤
   → 相差≥2个星级直接排除

3️⃣ 第三层：IQR异常值过滤 🆕
   → Q1 - 1.5×IQR < 价格 < Q3 + 1.5×IQR
   → 自动排除统计异常值

4️⃣ 第四层：多维加权评分
   → 支持动态权重配置，按场景预设：
   • downtown（市中心）: distance 40%
   • resort（度假景区）: star 40%
   • price_battle（价格战）: price 40%

5️⃣ 加分项：Occupancy信号 🆕
   → 竞品满房 +10分
   → 竞品高入住 +5分

6️⃣ 第五层：TopN精选 🆕
   → 只保留得分最高的Top 3-8家竞品
   → 避免过多竞品稀释核心信号
```

> 瑞湾平日价仅为豪华型竞品均值865元的41%，导致同等涨幅下绝对价差距被放大。五一定价应以绝对值校准为主，目标：比最近竞品（希尔顿欢朋920元）便宜150-200元，维持性价比优势。

这样的定性洞察是基于CPI分析得出：
- CPI = 443 / ((916+848+721+673)/4) × 100 = **56.9**
- CPI < 70 → 经济型定位，性价比优势是核心竞争力

## 常见问题

**Q: 为什么有时候筛选出来的有效竞品很少？**
A: 这是正常的。严格分层过滤是为了保证竞品的可比性。如果周边真的没有同档次同距离酒店，筛选出来少反而结果更准确。

**Q: 动态权重怎么选择？**
A:
- 市中心酒店：客人对位置敏感 → downtown配置（距离权重高）
- 度假景区酒店：客人对星级品牌敏感 → resort配置（星级权重高）
- 价格战区域：客人对价格敏感 → price_battle配置（价格权重高）

**Q: IQR异常过滤会不会滤掉正确数据？**
A: IQR是统计上成熟的异常检测方法，只有真正极端的值才会被过滤。如果数据都正常，不会过滤。

**Q: 其他酒店可以直接用吗？**
A: 可以。只需要填基础信息（名称、品牌、星级、平日价、坐标、范围），剩下的全自动。
