# 竞品报告输出规范与验收标准

## 报告验收清单

### 核心数据完整性
- [ ] 竞品数量 ≥ 目标数量（默认5家）
- [ ] 每家竞品含：名称/星级/假期价/平日价/涨幅
- [ ] 竞品分档：豪华型 / 高档型 / 舒适型
- [ ] 涨幅计算正确：`(假期价-平日价)/平日价 × 100%`

### 分析结构完整性
- [ ] 核心发现章节（竞品对比表 + 关键洞察）
- [ ] 定价建议章节（三档策略 + 房型分级）
- [ ] 行动建议章节（立即行动 + 持续监测）
- [ ] 数据附录章节（竞品详情 + 算法参数）
- [ ] 方法论沉淀（SOP + 注意事项）

### 双重校准算法验收
- [ ] 涨幅校准区间 = 中位数 ±15%
- [ ] 绝对值校准上限 = 竞品最高价 × 0.95
- [ ] 三档策略：保守 ≤ 标准 ≤ 激进
- [ ] 推荐标准档涨幅贴近竞品中位数

### 输出文件完整性
- [ ] `*_竞品价格分析报告.md` 存在
- [ ] `*_竞品价格分析报告.html` 存在
- [ ] `*_pricing_recommendation.json` 存在
- [ ] `*_raw_data.json` 存在

### 格式规范
- [ ] Markdown 使用标准标题层级（H1/H2/H3）
- [ ] 表格有表头，表头样式正确
- [ ] 重点数据加粗（建议价格/涨幅）
- [ ] 报告末尾有方法论沉淀章节

## JSON 输出规范

```json
{
  "hotel": "酒店名称",
  "target_date": "2026-05-01",
  "base_price": 443,
  "recommendation": {
    "conservative": {"price": 550, "rate": 24.2},
    "standard": {"price": 620, "rate": 40.0, "recommended": true},
    "aggressive": {"price": 710, "rate": 60.3}
  },
  "calibration": {
    "median_rate": 39.0,
    "range": {"lower": 530, "upper": 680},
    "abs_upper": 1327,
    "abs_lower": 420
  }
}
```

## 质量红线（不合格项）

以下任一情况需返工：
1. 竞品数量 < 3家
2. 涨幅计算错误
3. 缺少三档策略
4. 推荐价格超出绝对值上限
5. 无方法论沉淀章节
