# 酒店竞品调研 SOP（标准作业流程）

## 适用场景

- 五一、国庆、春节等假期定价决策
- 新开业酒店市场定位
- 收益管理策略制定
- 竞品监控与预警

## 调研范围定义

### 滨海核心区（直接竞品）
- 地理范围：纬度 38.9-39.1，经度 117.6-117.8
- 星级：4-5星豪华型
- 距离：5km 以内
- 代表：于家堡洲际、滨海皇冠假日、泰达万豪

### 市区高端（间接竞品/溢出客群）
- 地理范围：天津市区（和平/河东/河西）
- 星级：5星豪华型
- 定位：吸引滨海→市区溢出的高端客群

## 数据采集规范

### 飞猪（Fliggy）
```
工具：FlyAI MCP（Fliggy 实时 API）
优势：滨海新区数据全，实时价格更新
调用方式：flyai mcp call fliggy search_hotel --hotel <城市> --date <日期> --limit <数量>
```

### 携程（CTrip）
```
工具：携程开放平台 API 或爬虫
优势：数据最全，覆盖市区高端
注意：需API权限，建议申请企业账号
```

### 美团
```
工具：美团酒店开放平台
优势：中端市场数据补充
用途：验证涨幅趋势
```

## 数据字段规范

每条竞品数据必须包含：
- `name`：酒店全称
- `star`：星级（豪华型/高档型/舒适型）
- `base`：平日价格（人民币）
- `holiday`：假期价格（人民币）
- `rate`：涨幅百分比
- `distance`：距我店距离或行政区
- `decoration`：最近装修年份
- `source`：数据来源（fliggy/ctrip/meituan）
- `collected_at`：采集时间戳

## 质量校验

- [ ] 每家竞品至少2个数据源交叉验证
- [ ] 价格差异超过20%需人工复核
- [ ] 涨幅超过100%视为异常值需剔除
- [ ] 装修年份超过10年需标注"老旧"

## 报告产出标准

- Markdown 报告（主文件）
- HTML 报告（方便分享）
- JSON 建议（供 PMS 对接）
- 原始数据存档（JSON）

## 更新频率

- 假期前30天：每日更新
- 假期前7天：每半日更新
- 假期期间：实时监控
